      • Q: What is grid-based design?
      • A: Grid-based design breaks up a section of the webpage into configurable columns which we can use to alter layout. For example we can group the 12 columns into 2 usable columns of 6 grouped cells, thus dividing the screen in half. We can then use each of the two resulting columns to add photos & media in order to group content.

      • Q: What's all the hype about responsive webpages?'
      • A: These allow our webpages to reconfigure themselves on devices with differing screen sizes, so that the content remains clear and essential functions remain accessible (eg the hamburger button on headers to access the navigation bar).

      • Q: What is semantic structure?
      • A: Semantic structure is about ensuring our html is grouped into tags that are relevant to other coders when working with our code. For example an html doc with only div tags for content has no semantic structure so when we look at the code in inspector, we can't easily determine a structure. However if we change our blog tags to show header, main, section, div, and footer tags, then it's much easier to understand the layout of the code and work with it.
