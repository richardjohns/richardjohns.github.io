      Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

      I would say that HTML is a language composed of elements, used by the web-brower to structure the content of a webpage.

      CSS is another language used to style the way the html elements are displayed on-screen.

      Then the Document Object Model (DOM) essentially takes the html elements and turns them into javascript objects that we can manipulate or make interactive. The browser itself uses the DOM to take the html elements and display them on-screen. To put it another way, it's the interface between our html, javascript, and css. Also it's called a document model because every element is within the html document ie '!DOCTYPE html'

      We could use an analogy of a building to describe this, in that DOM is the undeerlying structure that holds the building together eg foundations and support beams, while HTML is the materials used ie concrete, wood, steel, while CSS represents how the building is styled architecturally or aesthetically to give it some appeal.

      What is meant by boxifying design?

      In order to structure the way we display content on a webpage, we need to layout the content into boxes, since html relies on the box model ie only allows content to be laid out in squares (including squares inside other squares).

      What is the box model?

      Each box within an html document has the content which has a specific height and width dimension. However around the content are the additional layers which are margin, border, and padding... all of which can be manipulated using CSS.
