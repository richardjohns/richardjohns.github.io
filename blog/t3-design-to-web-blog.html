      What a responsive site is, and why responsiveness is important?

      Responsive sites are sites that adapt their layout and content to the size of the screen they're being displayed on. It's simply important in order to maintain accessibility and functionality, and clarity of message to the user across a variety of devices.

      What is mobile-first design, and why is it important?

      Mobile first design is where the site is designed for small mobile size screens first, and then media queries are used to change content (using min-width css) when the site is accessed on a screen that's above a certain size. This helps smaller and less powerful devices to load the page more quickly as they don't have to run all css before the relevant css is accessed.

      What are frameworks, listing their pros and cons.

      Frameworks such bootstrap and foundation deal with a lot of complex, relsponsive website elements including the grid system using classes linked to javascript. This can make website development really easy and quick, but it can limit flexibility and creativity.

      What is a wireframe, and why do we use it?

      Wireframes allow us to plan in advance our website development on the html and css sides which means we're more likely to develop the site quickly and acheive the desired webpage.

      The aspects of your wireframes you found difficult to implement, and why.

      Acheiving aspects of my wireframe design presented a number of difficulties, primarily with dealing with flexbox layouts for the blog headlines on my index page. Another problem has been properly centering, sizing, and moving images in a way that still works in a responsive way between different screen sizes - still not there with that! For some reason "center" properties don't really wind up centered, and the skeleton buttons don't respond to having their css position data changed. Finally, the use of background images for my blog thumbnails has resulted in massively long page load times!
